О документе: описывает тестовый порядок запуска проекта.

Принятое допущение: точные команды запуска нужно сверить с репозиторием. Ниже тестовый вариант.

## 1. Предусловия

Для запуска нужны:

- Node.js;
- Python;
- зависимости frontend;
- зависимости backend;
- доступ к тестовой БД / 1С-контуру;
- тестовый набор лидов.

## 2. Запуск frontend

```bash
npm install
npm run dev
```

## 3. Запуск backend

```bash
pip install -r requirements.txt
uvicorn main:app --reload
```

## 4. Проверка

1. открыть frontend;
2. выбрать роль;
3. проверить список лидов;
4. создать лида;
5. проверить API;
6. выполнить импорт;
7. выполнить экспорт;
8. открыть отчёт.

## 5. Возможные проблемы

|**Проблема**|**Причина**|**Решение**|
|---|---|---|
|лиды не отображаются|backend недоступен|проверить запуск API|
|импорт не работает|неверный формат файла|проверить CSV/XLSX|
|дубли появляются|не работает проверка `lead_uid`|проверить импорт|
|роль не сохраняется|проблема cookie|проверить js-cookie|
|история пустая|нет записей `leads_stage`|проверить создание событий|
