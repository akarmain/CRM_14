## 2. Тестовые сценарии

### T1. Менеджер создаёт и ведёт лида

Шаги:

1. выбрать роль `manager_1`;
2. открыть таблицу;
3. нажать «Добавить лид»;
4. заполнить поля;
5. сохранить;
6. перевести лид в `qualified`;
7. открыть историю.

Ожидаемый результат:

- лид виден менеджеру;
- текущая стадия обновлена;
- история содержит `new` и `qualified`.

### T2. Аналитик смотрит данные без редактирования

Шаги:

1. выбрать роль аналитика;
2. открыть таблицу;
3. открыть канбан;
4. открыть историю лида;
5. попробовать изменить стадию.

Ожидаемый результат:

- аналитик видит данные;
- кнопки редактирования недоступны;
- изменение стадии невозможно.

### T3. РОП одобряет возврат

Шаги:

1. менеджер создаёт запрос возврата;
2. РОП открывает очередь;
3. РОП одобряет возврат.

Ожидаемый результат:

- стадия изменяется назад;
- история обновляется.

### T4. Импорт с дублями

Шаги:

1. загрузить файл с лидами;
2. загрузить тот же файл повторно.

Ожидаемый результат:

- при первой загрузке лиды добавлены;
- при второй загрузке дубли по `lead_uid` пропущены.
