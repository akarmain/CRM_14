# DOC-ARC-002 Контекст системы и границы решения

|**Версия**|**Статус**|**Дата создания**|**Дата обновления**|
|---|---|---|---|
|v0.2-test|Draft|2026-04-27|2026-04-27|

О документе: описывает контекст CRM_14, компоненты и внешние границы.

## 1. Контекст

CRM_14 состоит из:

- frontend-приложения;
- backend API;
- учебного контура данных;
- 1С-компонента;
- механизма импорта/экспорта;
- аналитического расчёта KPI.

## 2. Компоненты

|**Компонент**|**Назначение**|
|---|---|
|Frontend|интерфейс ролей, таблицы, канбан, формы, история|
|Backend|API, бизнес-правила, KPI, обработка данных|
|1С-контур|учебная база/таблицы и HTTP-методы связи|
|Логическая БД|модель `mini_crm_simple`|
|Импорт|загрузка CSV/XLSX|
|Экспорт|выгрузка Excel/CSV|
|Аналитика|конверсия, длительность стадий, отчёт|

## 3. Границы MVP

Внутри границ:

- лиды;
- стадии;
- владельцы;
- источники;
- история стадий;
- комментарии;
- импорт;
- экспорт;
- KPI.

Вне границ:

- телефония;
- email;
- авторизация;
- персональные данные;
- финансы;
- полноценная CRM;
- внешние интеграции;
- production deployment.

## 4. Упрощения

- роль выбирается на стартовом экране;
- пользователи в БД представлены enum;
- источники представлены enum;
- тестовые данные искусственные;
- права доступа реализуются на уровне интерфейса и backend-логики;
- логическая БД описана как PostgreSQL, а 1С используется как учебный контур.
