# DOC-GOV-004 Ретроспектива команды и lessons learned

|**Версия**|**Статус**|**Дата создания**|**Дата обновления**|
|---|---|---|---|
|v0.2-test|Draft|2026-04-27|2026-04-27|

О документе: фиксирует предварительную ретроспективу команды по итогам тестовой версии документации.

Для кого: для команды и преподавателя.

## 1. Что получилось хорошо

Команда смогла собрать понятный учебный MVP, в котором есть не только интерфейс, но и логика движения лида, базовая модель данных, роли пользователей и аналитика по воронке.

Сильная сторона проекта — наличие демонстрационных экранов для разных ролей:

- менеджер;
- аналитик;
- руководитель отдела продаж.

Это позволяет показать не просто таблицу лидов, а различия в правах и задачах пользователей.

## 2. Что оказалось сложным

Сложной частью стала стыковка нескольких контуров:

- frontend;
- backend;
- логическая БД;
- 1С;
- импорт/экспорт;
- права ролей.

Также сложность возникла из-за учебной авторизации через выбор роли. С одной стороны, это удобно для демо. С другой стороны, в документации нужно явно писать, что это не промышленная безопасность.

## 3. Что нужно улучшить

- выровнять даты проекта;
- уточнить фактические endpoints API;
- проверить, какие функции реально реализованы;
- уточнить, где именно хранится основной набор данных: 1С, backend или тестовая БД;
- добавить реальные скриншоты и схемы;
- заменить текстовую ERD на изображение или dbdiagram;
- уточнить пороги KPI;
- добавить финальный demo walkthrough.

## 4. Главный урок

Даже мини-CRM требует чёткого описания бизнес-правил. Если не зафиксировать порядок стадий, правила возврата и права ролей, система быстро становится просто таблицей лидов без управляемого процесса.
