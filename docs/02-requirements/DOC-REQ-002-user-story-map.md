# DOC-REQ-002 User Story Map

|**Версия**|**Статус**|**Дата создания**|**Дата обновления**|
|---|---|---|---|
|v0.2-test|Draft|2026-04-27|2026-04-27|

О документе: описывает пользовательский путь CRM_14 от выбора роли до анализа и экспорта данных.

## 1. Backbone пользовательского пути

|**Шаг**|**Менеджер**|**Аналитик**|**РОП**|
|---|---|---|---|
|1. Выбор роли|выбирает менеджера|выбирает аналитика|выбирает РОП|
|2. Обзор данных|видит свои лиды|видит все лиды|видит все лиды|
|3. Работа со списком|создаёт и ведёт лиды|смотрит и фильтрует|управляет всеми лидами|
|4. Канбан|переводит свои лиды|анализирует стадии|контролирует стадии|
|5. История|смотрит свои лиды|смотрит историю|смотрит историю|
|6. Возврат|запрашивает возврат|нет действия|одобряет/отклоняет|
|7. Аналитика|нет доступа|смотрит KPI|смотрит KPI|
|8. Импорт/экспорт|импорт в рамках сценария|экспорт|импорт и экспорт|

## 2. MVP-срез

В MVP входят:

- выбор роли;
- список лидов;
- канбан;
- создание лида;
- изменение статуса;
- история лида;
- запрос возврата;
- одобрение возврата;
- импорт;
- экспорт;
- базовые KPI.

## 3. Следующие версии

После MVP можно добавить:

- полноценную авторизацию;
- таблицу пользователей;
- справочник источников;
- таблицу заявок на возврат;
- расширенные визуализации;
- теги;
- сумму сделки;
- причины проигрыша;
- интеграцию с телефонией;
- email-коммуникации.
