О документе: описывает пользователей CRM_14, их задачи, боли, доступные действия и ограничения.

Для кого: для аналитика, frontend, backend и преподавателя.

## 1. Подход к пользователям

В проекте не создаются подробные вымышленные персоны с именами. Вместо этого используются роли, которые уже есть в демо и бизнес-логике проекта:

- менеджер;
- аналитик;
- руководитель отдела продаж.

В тестовых данных есть два менеджера: `manager_1` и `manager_2`.

## 2. Роль: Менеджер

Основная задача: операционная работа с лидами и их статусами.

### Контекст

Менеджер получает или создаёт лиды, ведёт их по стадиям и фиксирует изменения. Он отвечает только за свои лиды и не должен видеть чужую работу.

### Действия

Менеджер может:

- создать лид вручную;
- создать лид через импорт;
- посмотреть список своих лидов;
- посмотреть канбан по своим лидам;
- перевести лид по стадиям вперёд;
- инициировать возврат на предыдущую стадию;
- заполнить комментарий;
- посмотреть историю стадий своих лидов.

### Права

Менеджер имеет доступ:

- к изменению статусов в рамках бизнес-правил;
- к просмотру только своих лидов;
- к просмотру истории своих лидов;
- к запросу возврата стадии.

Менеджер не имеет доступа:

- к аналитике;
- к просмотру чужих лидов;
- к редактированию чужих лидов;
- к одобрению возврата стадии.

### Боли менеджера

- сложно быстро понять, какие лиды сейчас требуют действия;
- неудобно вести статусы вручную;
- можно забыть, почему лид был переведён на определённую стадию;
- без истории сложно восстановить контекст работы.

### Цифровые навыки

Менеджер уверенно работает с простыми веб-интерфейсами, таблицами и фильтрами. Ему не нужны сложные настройки, но важны понятные кнопки и быстрый доступ к своим лидам.

## 3. Роль: Аналитик

Основная задача: расчёт и валидация метрик, анализ качества данных.

### Контекст

Аналитик не ведёт лиды напрямую. Он смотрит на данные по всей воронке, анализирует конверсию и длительность стадий, выявляет узкие места и готовит выводы.

### Действия

Аналитик может:

- просматривать всю воронку;
- просматривать список лидов;
- смотреть канбан;
- открывать историю лида;
- рассчитывать конверсию;
- рассчитывать среднюю длительность стадий;
- экспортировать данные для анализа.

### Права

Аналитик имеет доступ:

- к просмотру всех лидов;
- к просмотру истории;
- к аналитическим данным;
- к экспорту.

Аналитик не имеет доступа:

- к созданию лидов;
- к редактированию лидов;
- к изменению статусов;
- к одобрению возврата стадий.

### Боли аналитика

- сложно понять, на какой стадии воронка теряет больше всего лидов;
- нужно сравнивать источники и менеджеров;
- нужно быстро получать данные для отчёта;
- важно видеть длительность стадий без ручного расчёта.

### Цифровые навыки

Аналитик уверенно работает с таблицами, фильтрами, экспортами и метриками. Ему важно видеть корректные данные и понимать, как рассчитаны KPI.

## 4. Роль: Руководитель отдела продаж

Основная задача: контроль конверсии, эффективности менеджеров и источников, управление процессом продаж.

### Контекст

РОП отвечает за общую прозрачность продаж. Он видит всех лидов, контролирует работу менеджеров, проверяет воронку и принимает решения по процессу.

### Действия

РОП может:

- просматривать всю воронку;
- просматривать список всех лидов;
- создавать лиды;
- редактировать любые лиды;
- переводить лиды по стадиям;
- фильтровать по менеджеру;
- фильтровать по источнику;
- фильтровать по периоду;
- экспортировать отчёт;
- контролировать историю изменений;
- одобрять возврат лида на предыдущую стадию;
- отклонять возврат;
- принудительно менять стадию при необходимости.

### Права

РОП имеет полный доступ:

- просмотр всех лидов;
- создание лидов;
- редактирование любых лидов;
- перевод по стадиям;
- одобрение возврата стадии назад;
- просмотр истории стадий;
- экспорт данных как Excel и CSV;
- импорт данных;
- просмотр логов изменений.

### Боли РОП

- сложно контролировать работу менеджеров без единой воронки;
- нужно видеть, где лиды застревают;
- важно понимать эффективность источников;
- нужно проверять спорные возвраты стадий;
- необходимо быстро получить отчёт для обсуждения.

### Цифровые навыки

РОП умеет работать с CRM-подобными интерфейсами, таблицами, фильтрами и отчётами. Ему важна не техническая детализация, а быстрый управленческий обзор.

## 5. Различия ролей

|**Возможность**|**Менеджер**|**Аналитик**|**РОП**|
|---|---|---|---|
|Видит свои лиды|Да|Да|Да|
|Видит все лиды|Нет|Да|Да|
|Создаёт лиды|Да|Нет|Да|
|Меняет стадии|Да, только свои и по правилам|Нет|Да|
|Запрашивает возврат|Да|Нет|Да|
|Одобряет возврат|Нет|Нет|Да|
|Смотрит аналитику|Нет|Да|Да|
|Экспортирует отчёт|Нет / ограниченно|Да|Да|
|Импортирует данные|Да / через сценарий|Нет|Да|
|Редактирует чужие лиды|Нет|Нет|Да|

Принятое допущение: в уточнениях менеджер может создавать лид через импорт, а РОП имеет полный импорт. Если в финальной реализации импорт доступен только РОП, это нужно поправить.
