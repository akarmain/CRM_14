Ты — AI backend-инженер. Твоя задача — спроектировать и реализовать API для Telegram Mini App «Контактная книжка» на основе описанного UI.

ОПИСАНИЕ ПРОДУКТА
Приложение — персональная контактная книжка (один пользователь).
Основной экран — список контактов.
При нажатии на контакт открывается экран редактирования/просмотра контакта.

ОПИСАНИЕ ЭКРАНОВ (для понимания логики API)

1) Экран списка контактов:
- список карточек
- каждая карточка содержит:
  - avatar (круг)
  - ФИО (одной строкой)
- сверху поиск
- снизу кнопка «Добавить»  (Telegram MainButton)

2) Экран контакта:
- avatar (круг)
- ФИО + дата рождения
- при нажатии на аватар появляется строка для ввода ссылки 
- Поле ввода для ввода ключа и значения 
- снизу две кнопки «Сохранить»   «Удалить»  (Telegram MainButton)

ТРЕБОВАНИЯ К API

Реализовать REST API v1 со следующими эндпойнтами:

1. GET /api/v1/contacts
   - возвращает список всех контактов
   - поддерживает query-параметр search (поиск по ФИО и дате рождения в формате ДДММГГГГ)

2. GET /api/v1/contacts/{id}
   - возвращает один контакт

3. POST /api/v1/contacts
   - создаёт новый контакт
   - принимает обязательные поля ФИО
   - остальные поля опциональны

4. PUT /api/v1/contacts/{id}
   - обновляет контакт целиком

5. PATCH /api/v1/contacts/{id}
   - частичное обновление (например, avatar_url)

6. DELETE /api/v1/contacts/{id}

ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ
- FastAPI
- Pydantic schemas (Create / Update / Read)
- SQLAlchemy 2.0 (async)
- SQLite (один пользователь)
- extra_fields хранить в JSON колонке
- Чёткое разделение слоёв:
  - API (routers)
  - Service
  - Repository
  - DB models
- CORS включён для Telegram Mini App
- Добавь пример seed-данных (2–3 контакта)

Ты — AI frontend-инженер. Твоя задача — реализовать интерфейс Telegram Mini App (TMA) по описанному макету.

СТЕК
- Nuxt 4
- TypeScript
- Telegram WebApp SDK
- Минимальный UI (без лишней стилизации)
- Mobile-first (вертикальный экран)

ОПИСАНИЕ UI (по изображению)

=== ЭКРАН 1: Список контактов ===

Верх:
- Поле поиска (input)
- Поиск фильтрует список по ФИО и дате рождения

Центр:
- Вертикальный список карточек контактов
- Карточка контакта:
  - слева: круглый аватар
  - справа:
    - ФИО (жирно)
- При нажатии на карточку → переход на экран контакта

Низ:
- Telegram MainButton с текстом «Добавить»
- При нажатии → создание нового контакта и переход на экран контакта

=== ЭКРАН 2: Контакт ===

Верх:
- Круглый аватар
- Текст:
  - ФИО
  - дата рождения (если есть)

Центр:
  поля для воода ключей и занчнеий которые задете непосредстенно пользователь

Низ 2 кнопки Telegram MainButton :
- Telegram MainButton с текстом «Сохранить»
- При нажатии:
  - PATCH /contacts/{id}
  - сохранение avatar_url
  - возврат на экран списка


- Telegram MainButton с текстом «удолить»
